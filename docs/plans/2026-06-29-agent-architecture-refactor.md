# agent-architecture Package Refactor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split the monolithic `agent-architecture` npm package into a slim core + three optional `@agent-arch/*` scoped add-on packages, with zero breaking changes to the install CLI.

**Architecture:** `agent-architecture` becomes the workspace root containing core runtime, core agents (swe/qa-agent/spec-agent/pm), and ~25 universal skills. Three workspace sub-packages (`packages/adapters`, `packages/stacks`, `packages/skills`) are published independently as `@agent-arch/adapters`, `@agent-arch/stacks`, `@agent-arch/skills`. The installer auto-discovers any installed `@agent-arch/*` packages in `node_modules/@agent-arch/` and merges their skills into the install output.

**Tech Stack:** Node.js ESM, npm workspaces, Node built-in test runner (`node:test`), existing `scripts/install.mjs` installer pattern.

## Global Constraints

- `npx agent-architecture install` CLI command must not change.
- Package name `agent-architecture` must not change.
- `.agent/` output directory structure must not change.
- `install-manifest.json` schema must not change.
- All existing tests must pass after each task (fix tests when paths move).
- No new runtime dependencies — use only `node:fs`, `node:path`, `node:child_process` already in install.mjs.
- Use `git mv` (not `cp` + `rm`) for all directory moves to preserve git history.
- Working directory for all `git mv` commands: `agent-architecture/` subdirectory.
