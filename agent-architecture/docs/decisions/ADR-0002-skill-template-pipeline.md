# ADR-0002 — Skill Template Pipeline (tmpl → generated → installed)

**Status:** Accepted  
**Date:** 2026-06-25  
**SPEC refs:** C1, I1, V4, V13

## Context

Skills need to be:
- Editable by contributors without regenerating every time.
- Consistent across multiple host targets (Claude, Codex, Copilot, Google ADK, Strands).
- Verifiable (CI can catch stale generated files).
- Installable in repos that do not run the generator (offline, enterprise-restricted).

A single static Markdown file per skill is easy to edit but hard to keep consistent
across hosts. A fully dynamic generator that runs at install time requires a Node.js
runtime in every consumer repo.

## Decision

Skills use a two-level pipeline:

```
<skill>/SKILL.md.tmpl   (source of truth — edit this)
       ↓  npm run build:skills
<skill>/SKILL.md        (generated — committed alongside tmpl)
       ↓  npm run install (or npx agent-architecture install)
.agent/<skill>/SKILL.md (installed into consuming repo)
```

Template placeholders (`{{PREAMBLE}}`, `{{POLICY_REQUIREMENTS}}`, `{{BASE_OUTPUT_RULES}}`)
are resolved by `scripts/gen-skill-docs.mjs` against a fixed resolver table. No
network calls, no dynamic data.

Generated files are committed so:
- Consuming repos can install without running the generator.
- CI verifies freshness via `npm run check:skills --check`.
- PRs show both the template diff and the generated diff.

Host-specific artifacts (Codex `AGENTS.md`, Copilot `copilot-instructions.md`) are
generated by a second pipeline (`scripts/gen-host-artifacts.mjs`) from the root
`SKILL.md` and the `agents/` directory. Same pattern: committed, CI-verified via
`npm run check:hosts`.

## Consequences

- Contributors edit `.tmpl` files, run `npm run build:skills`, and commit both.
- Merge conflicts on generated files are resolved by fixing the template and
  re-running the generator (never accepting one side's generated output).
- Adding a new host target requires adding an output function in
  `gen-host-artifacts.mjs` and registering in `hosts/registry.json` — no skill edits.
- Skills that diverge between tmpl and generated are caught by CI before merge.
