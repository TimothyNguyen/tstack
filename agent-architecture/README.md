## Package Structure

This repo uses an npm workspace split: a core package plus three optional add-on packages.

### Core package (this directory root)

Always installed. Contains:

| Path | What's here |
|---|---|
| `agents/` | Role-based agent skills (swe, qa-agent, pm, spec-agent, orchestrate, …) |
| `autoplan/`, `brainstorming/`, `canary/`, `careful/`, `commit/`, … | Core skills shipped with every install |
| `bin/` | CLI entry point (`agent-architecture` command) |
| `scripts/` | Build, generate, validate, install scripts |
| `core/` | Subagent orchestration, worktree, event primitives |
| `hooks/` | Hook runners for Claude, Codex, Cursor |
| `generated/` | Registry + skills index (committed, regenerated by `build:skills`) |
| `hosts/` | Per-host config registry |
| `policies/` | Enterprise default policy JSON |
| `tests/` | Main test suite (543 tests) |
| `docs/` | Package-level reference docs |

### Add-on packages (`packages/`)

Optional npm workspaces. Install independently or via the installer's add-on discovery.

#### `packages/skills/` — specialty skills with code

| Package | Code |
|---|---|
| `codebase-engine/` | Python graph extraction engine + 100+ tests (`codebase_engine/*.py`) |
| `atlassian-docs/` | Jira, Confluence, Bitbucket MCP servers (`mcp/src/*_mcp_server/`) |
| `drawio-mcp-python/` | draw.io MCP server (`drawio_mcp/server.py`) |
| `diagram-cloudformation/` | CloudFormation diagram MCP (`server.py`) |
| `diagram-helm/` | Helm chart diagram MCP (`server.py`) |
| `diagram-iac/` | IaC diagram MCP (`server.py`) |
| `diagram-infrastructure/` | Infrastructure diagram MCP (`server.py`) |
| `seniorswe-concise/` | Anti-bloat skill suite (6 sub-skills) |
| `benchmark/`, `chrome-devtools/`, `documentation/`, … | Other specialty skills |

#### `packages/adapters/` — framework adapters

| Package | What it does |
|---|---|
| `adapter-ag-ui/` | AG-UI protocol adapter |
| `adapter-agentcore/` | AWS AgentCore adapter |
| `adapter-databricks/` | Databricks adapter |
| `adapter-docker-mcp/` | Docker MCP adapter |
| `adapter-github/` | GitHub adapter |
| `adapter-google-adk/` | Google ADK adapter |
| `adapter-langgraph/` | LangGraph adapter |
| `adapter-mcp/` | Generic MCP adapter |
| `adapter-openapi/` | OpenAPI adapter |
| `adapter-seniorswe-concise/` | seniorswe-concise runtime adapter (hooks + MCP) |
| `adapter-strands/` | Strands adapter |

#### `packages/stacks/` — tech/domain stacks

AWS, DMS, C#, Databricks, DBT, legacy frontend, Postgres, Python, React/TypeScript, Spring AI, Spring Boot, SQL Server, SQL Server→Postgres migration, domain ML/data stacks.

## Core Commands

```bash
npm run build:skills    # regenerate all skill docs + registry
npm run check:skills    # verify nothing is stale
npm run gen:hosts       # regenerate CLAUDE.md / AGENTS.md / copilot-instructions.md
npm test                # run full test suite
```

## Repo Shape (quick map)

```
agent-architecture/
├── agents/             # role agents (swe, qa-agent, pm, …)
├── <skill>/            # core skills (autoplan, commit, guard, …)
├── bin/                # CLI
├── core/               # subagent primitives
├── hooks/              # hook runners
├── scripts/            # build + generate scripts
├── generated/          # committed registry + index
├── tests/              # test suite
├── packages/
│   ├── skills/         # add-on skills WITH code (codebase-engine, atlassian, drawio, diagram-*)
│   ├── adapters/       # framework adapters
│   └── stacks/         # tech/domain stacks
└── docs/               # reference docs
```
