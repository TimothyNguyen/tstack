# Repository Change Governance Workflow

This file used to hold a very large governance policy draft.

Current repo policy is simpler:

1. Keep repo entrypoints obvious.
2. Generate governance inventory with `npm run governance:build`.
3. Validate with `npm run governance:check`.
4. Commit generated governance artifacts when component inventory changes.
5. Use [GOVERNANCE_AUTOMATION.md](GOVERNANCE_AUTOMATION.md) as source of truth.
